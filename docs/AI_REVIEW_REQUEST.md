# Revisão Arquitetural do Mouse IA

Você deverá realizar uma revisão completa do estado atual do projeto antes de qualquer nova implementação.

## Objetivo

Garantir que o código existente esteja alinhado com a arquitetura oficial documentada.

## Documentação obrigatória para leitura

Leia primeiro:

- docs/ARCHITECTURE.md
- docs/AGENT.md
- docs/DECISIONS.md
- docs/ROADMAP.md
- docs/SECURITY.md

Depois leia:

- tasks/v0.1/001-bootstrap.md
- tasks/v0.1/002-authentication.md
- tasks/v0.1/003-organizations.md
- tasks/v0.1/004-projects.md
- tasks/v0.1/005-assets.md

## Processo de revisão

Faça uma auditoria comparando:

Arquitetura esperada

versus

Implementação atual.

Analise:

- Backend;
- Frontend;
- Banco de dados;
- Modelos;
- APIs;
- Services;
- Repositories;
- Componentes React;
- Configurações;
- Dependências.

## Identifique:

1. Funcionalidades já implementadas;
2. Funcionalidades parcialmente implementadas;
3. Funcionalidades ausentes;
4. Código fora da arquitetura definida;
5. Código que precisa ser refatorado;
6. Possíveis problemas de segurança;
7. Divergências entre documentação e implementação.

## Regras importantes

Não altere nenhum arquivo ainda.

Primeiro produza um relatório contendo:

# Estado Atual

O que existe hoje.

# Divergências Arquiteturais

O que está diferente da documentação.

# Riscos Técnicos

Problemas encontrados.

# Plano de Ajustes

Ordem recomendada de correção.

# Impacto

Classifique cada item:

- Baixo
- Médio
- Alto
- Crítico

Somente após minha aprovação você deverá modificar arquivos.

---

Ao final, confirme:

"Revisão concluída. Aguardando autorização para aplicar alterações."
